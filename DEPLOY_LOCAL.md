# 🌐 Развёртывание во внутренней сети

## Инструкция по настройке доступа для других пользователей

Эта инструкция поможет вам настроить доступ к дашборду для других пользователей вашей локальной сети.

---

## 📋 Требования

- Windows 10/11
- Node.js 18.x или выше
- Статический IP-адрес в локальной сети (рекомендуется)
- Права администратора для настройки брандмауэра

---

## 🚀 Пошаговая инструкция

### Шаг 1: Узнайте ваш IP-адрес в локальной сети

Откройте командную строку и выполните:

```cmd
ipconfig
```

Найдите ваш **IPv4-адрес** (например, `192.168.1.100` или `10.0.0.5`).

> 💡 **Важно:** Если у вас динамический IP (DHCP), он может измениться после перезагрузки. Для постоянного доступа настройте статический IP в роутере.

---

### Шаг 2: Настройте переменные окружения

1. Скопируйте файл `.env.example` в `.env`:

```cmd
copy .env.example .env
```

2. Откройте `.env` и замените `192.168.1.100` на ваш IP-адрес:

```env
PORT=5000
HOST=0.0.0.0
ALLOWED_ORIGINS=http://localhost:3000,http://localhost:5173,http://192.168.1.100:3000,http://192.168.1.100:5173
CACHE_TTL=30000
LOG_LEVEL=info
```

---

### Шаг 3: Откройте порты в брандмауэре

#### Вариант A: Через командную строку (от администратора)

```cmd
netsh advfirewall firewall add rule name="Goszakupki Server" dir=in action=allow protocol=TCP localport=5000
netsh advfirewall firewall add rule name="Goszakupki Frontend" dir=in action=allow protocol=TCP localport=3000
```

#### Вариант B: Через интерфейс Windows

1. `Панель управления` → `Брандмауэр Защитника Windows`
2. `Дополнительные параметры` → `Правила для входящих подключений`
3. `Создать правило...` → `Для порта` → `Далее`
4. Протокол: `TCP`, Порты: `3000, 5000` → `Далее`
5. `Разрешить подключение` → `Далее`
6. Выберите все профили (Доменный, Частный, Публичный) → `Далее`
7. Имя: `Goszakupki Dashboard` → `Готово`

---

### Шаг 4: Установите зависимости

```cmd
npm install
```

---

### Шаг 5: Запустите приложение

#### Вариант 1: Одновременный запуск (рекомендуется)

```cmd
npm run dev
```

#### Вариант 2: Production-режим

1. Соберите frontend:
```cmd
npm run build
```

2. Запустите сервер:
```cmd
npm run server
```

3. Для раздачи собранного frontend используйте веб-сервер (IIS, Nginx, Apache) или простой HTTP-сервер:
```cmd
npx serve build -l 3000
```

---

### Шаг 6: Проверьте доступ

1. **Локально:** Откройте в браузере
   - http://localhost:3000
   - http://localhost:5000/api/health

2. **По IP:** Откройте в браузере
   - http://192.168.1.100:3000 (замените на ваш IP)
   - http://192.168.1.100:5000/api/health

3. **С другого компьютера:** Откройте в браузере
   - http://192.168.1.100:3000 (IP компьютера с сервером)

---

## 🔗 URL для доступа пользователей

| Компонент | URL |
|-----------|-----|
| **Дашборд** | `http://192.168.1.100:3000` |
| **API** | `http://192.168.1.100:5000/api/data` |
| **Health check** | `http://192.168.1.100:5000/api/health` |

> Замените `192.168.1.100` на IP-адрес вашего компьютера.

## 🔧 Настройка для production

### Изменение конфига Vite

Для production-сборки обновите `vite.config.js`:

```javascript
server: {
  port: 3000,
  host: '0.0.0.0', // Разрешить подключения из сети
  proxy: {
    '/api': {
      target: 'http://localhost:5000',
      changeOrigin: true,
    },
  },
},
```

### Переменные окружения для production

Создайте `.env.production`:

```env
NODE_ENV=production
VITE_API_URL=http://192.168.1.100:5000
```

## 🛠️ Устранение проблем

### Ошибка: "Порт уже занят"

```cmd
# Найти процесс на порту
netstat -ano | findstr :3000
netstat -ano | findstr :5000

# Остановить процесс (замените PID)
taskkill /F /PID 12345
```

### Ошибка: "CORS origin not allowed"

Проверьте `.env` файл — добавьте IP вашего компьютера в `ALLOWED_ORIGINS`:

```env
ALLOWED_ORIGINS=http://localhost:3000,http://192.168.1.100:3000
```

### Брандмауэр блокирует подключение

1. Проверьте правила брандмауэра:
```cmd
netsh advfirewall firewall show rule name=all | findstr "Goszakupki"
```

2. Временно отключите брандмауэр для проверки:
```cmd
netsh advfirewall set allprofiles state off
```
> ⚠️ Не забудьте включить обратно!

### Подключение не проходит с другого компьютера

1. Проверьте, что оба компьютера в одной сети
2. Убедитесь, что IP не изменился (используйте `ipconfig`)
3. Проверьте антивирус — может блокировать порты
4. Попробуйте отключить VPN (если есть)

## 🔐 Безопасность

### Рекомендации для внутренней сети:

1. **Ограничьте доступ по IP** (в server.cjs):
```javascript
const allowedIPs = ['192.168.1.', '10.0.0.']; // Подсети
```

2. **Настройте HTTPS** (см. [HTTPS_SETUP.md](./HTTPS_SETUP.md))

3. **Используйте базовую аутентификацию** для защиты доступа

4. **Регулярно обновляйте зависимости**:
```cmd
npm audit
npm update
```

## 📊 Мониторинг

### Просмотр логов

Логи сервера сохраняются в папке `logs/`:
- `combined.log` — все логи
- `error.log` — только ошибки
- `web-vitals.jsonl` — метрики производительности

### Проверка статуса

```cmd
# Проверка работы API
curl http://localhost:5000/api/health

# Проверка данных
curl http://localhost:5000/api/data?limit=5
```

## 🔄 Автоматический запуск при загрузке Windows

### Вариант 1: Планировщик заданий

1. Откройте `Планировщик заданий`
2. `Создать задачу...`
3. Вкладка `Триггеры` → `При запуске`
4. Вкладка `Действия` → `Создать`
   - Программа: `C:\Program Files\nodejs\node.exe`
   - Аргументы: `C:\Users\Дмитрий\Dashboards\goszakupki_cgm\server.cjs`
   - Рабочая папка: `C:\Users\Дмитрий\Dashboards\goszakupki_cgm`

### Вариант 2: PM2 (рекомендуется)

```cmd
npm install -g pm2

pm2 start server.cjs --name goszakupki-api
pm2 save
pm2 startup
```

## 📞 Поддержка

При возникновении проблем:

1. Проверьте логи в `logs/error.log`
2. Проверьте консоль браузера (F12)
3. Убедитесь, что `database.xlsx` находится в корне проекта
4. Проверьте, что сервер запущен и порты открыты
