# Инструкция по запуску дашборда с автоматическим обновлением данных

## 🚀 Быстрый запуск

### Вариант 1: Запуск сервера и React приложения одновременно
```bash
npm run dev
```
Эта команда запустит:
- Сервер на порту 5000 (читает данные из Excel)
- React приложение на порту 3000

### Вариант 2: Раздельный запуск

**Терминал 1 - Сервер:**
```bash
npm run server
```

**Терминал 2 - React приложение:**
```bash
npm start
```

## 📊 Как это работает

1. **Сервер** (`server.js`) читает файл `database.xlsx` при каждом запросе
2. **Кэширование** данных на 30 секунд для производительности
3. **React приложение** загружает данные через API: `http://localhost:5000/api/data`
4. **Кнопка "Обновить данные"** в сайдбаре принудительно обновляет данные

## 🔄 Обновление данных

### Автоматически
1. Откройте `database.xlsx`
2. Внесите изменения
3. Сохраните файл (Ctrl+S)
4. Нажмите кнопку **"Обновить данные"** в дашборде
5. Данные обновятся автоматически!

### Через API
Сервер кэширует данные на 30 секунд. Для принудительного обновления:
- Нажмите кнопку "Обновить данные" в сайдбаре
- Или перезагрузите страницу браузера

## 🛠️ API Эндпоинты

| Эндпоинт | Описание |
|----------|----------|
| `GET /api/data` | Получить все данные из Excel |

## ⚙️ Настройки

### Порт сервера
Измените в `server.js`:
```javascript
const PORT = 5000; // Ваш порт
```

### Время кэширования
Измените в `server.js`:
```javascript
const CACHE_TTL = 30000; // 30 секунд (в миллисекундах)
```

## 🔧 Устранение проблем

### Сервер не запускается
```bash
# Проверьте, свободен ли порт 5000
netstat -ano | findstr :5000

# Остановите процесс, если занят
taskkill /F /PID <номер_процесса>
```

### Ошибка "Файл database.xlsx не найден"
Убедитесь, что файл находится в корне проекта:
```
C:\Users\Дмитрий\Dashboards\goszakupki_cgm\database.xlsx
```

### React не подключается к серверу
Проверьте CORS в браузере (F12 → Console). Ошибка CORS решается в `server.js`:
```javascript
app.use(cors());
```
